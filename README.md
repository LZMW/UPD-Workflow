# UPD Workflow v1.3

## 简介 / Introduction

一套融合深度推理机制的统一项目开发工作流规范，专为AI辅助开发设计。在成熟的6A5S框架基础上，创新性地集成了自研的深度推理机制，大幅提升AI在复杂项目中的推理质量和交付成功率。

A unified project development workflow specification integrating deep reasoning mechanisms, designed for AI-assisted development. Building upon the mature 6A5S framework, it innovatively incorporates a proprietary deep reasoning mechanism, significantly improving AI reasoning quality and delivery success rates in complex projects.

---

## 核心创新 / Core Innovation ⭐

### 深度推理机制（自研） / Deep Reasoning Mechanism (Proprietary)

这是本工作流的核心创新点，通过三步系统化思维流程，让AI在处理复杂任务时能够进行深度、结构化的推理。

This is the core innovation of this workflow. Through a three-step systematic thinking process, AI can perform deep, structured reasoning when handling complex tasks.

#### 三步执行流程 / Three-Step Execution Process

**1. 拆解 (Decompose)**
- 提取已知/未知/潜在需求
- Extract known/unknown/potential requirements
- 识别核心问题
- Identify core problems

**2. 解构 (Deconstruct)** - 并行三路思维 / Parallel Three-Way Thinking
- 可行性与知识回溯
- Feasibility and knowledge retrospection
- 执行方案与路径规划（≥3种）
- Execution solutions and path planning (>=3 options)
- 约束与边界设计（含否定约束）
- Constraints and boundary design (including negative constraints)

**3. 重组 (Reorganize)**
- 聚合评估
- Aggregate evaluation
- 择优路径
- Select optimal path
- 反思修正
- Reflect and revise
- 个性化输出
- Personalized output

#### 触发条件 / Trigger Conditions
- 复杂需求分析 / Complex requirement analysis
- 多方案决策 / Multi-option decisions
- 风险评估 / Risk assessment
- 创新方案 / Innovative solutions

#### 智能工具协作 / Intelligent Tool Collaboration
基于深度思考产出的上下文，智能路由至不同工具：

Based on context from deep thinking, intelligently route to different tools:

- **MCP sequential thinking** - 用于任务拆分、严密步骤规划、ToDoList生成 / For task breakdown, rigorous step planning, ToDoList generation
- **MCP Context7** - 用于查缺补漏、技术难题解决、最佳实践验证 / For gap filling, technical problem solving, best practice verification

## 6A5S 开发框架 / 6A5S Development Framework

本工作流基于成熟的 **6A5S 项目管理方法论**，融入了自研的深度推理机制，实现理论与实践的完美结合。

This workflow is based on the mature **6A5S Project Management Methodology**, integrating the proprietary deep reasoning mechanism for perfect combination of theory and practice.

### 6A 阶段 / 6A Phases

1. **Align (对齐)** - 模糊需求 → 精确规范 / Vague requirements → Precise specifications
2. **Architect (架构)** - 共识 → 架构设计 / Consensus → Architecture design
3. **Atomize (原子化)** - 设计 → 任务拆分 / Design → Task breakdown
4. **Approve (审批)** - 审查与确认 / Review and confirmation
5. **Automate (自动化执行)** - 按节点执行 → 测试 → 代码 → 文档 / Execute → Test → Code → Document
6. **Assess (评估)** - 结果评估 → 交付 / Result evaluation → Delivery

### 5S 规范 / 5S Standards

- 整理 / Sort - 明确需求边界 / Clarify requirement boundaries
- 整顿 / Set in Order - 规范文档结构 / Standardize document structure
- 清扫 / Shine - 质量检查 / Quality inspection
- 清洁 / Standardize - 流程标准化 / Process standardization
- 素养 / Sustain - 持续改进 / Continuous improvement

---

## 质量原则 / Quality Principles

1. **不延期** / No Delays - 严控风险，遇阻即调 / Strict risk control, adjust when blocked
2. **不超支** / No Overruns - 范围/资源严格匹配 / Strict scope/resource matching
3. **不出错** / No Errors - 多轮自检，出错即停 / Multi-round self-checking, stop on errors
